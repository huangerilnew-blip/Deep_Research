## 深度调研竞品洞察（Insights）

> 说明：本文件基于 `Competitor.md` 中列出的产品，并结合公开信息与行业常识，从以下六个维度梳理：
> 1）市场定位（目标用户、信息源、价值主张）
> 2）盈利方式
> 3）核心功能与差异点（含核心亮点、独特卖点与解决的痛点）
> 4）市场份额与增长趋势
> 5）服务质量
> 6）用户体验
> 若在调研中发现额外有价值的内容，会放入“其他信息”中。本文侧重于为 deepresearch 的产品战略与设计提供参考，而非严谨的行业报告。

---

## 一、场景划分与整体格局

### 1.1 按场景划分竞品

- **学术检索 / 学术问答 / 文献研究**：
  - Perplexity Pro（Deep Search/Focus，偏通用+科研）、Elicit、Semantic Scholar、Consensus、Google Scholar + 学术助手插件、Connected Papers、Research Rabbit、Scite
- **通用 AI 搜索与总结**：
  - Perplexity Pro、Kagi Universal Summarizer
- **企业情报 / 投研研究**：
  - CB Insights、AlphaSense
- **创投 / 公司数据库**：
  - PitchBook、Crunchbase
- **金融数据与机构终端（含中文）**：
  - FactSet、Refinitiv、Morningstar Direct、Wind、同花顺 iFinD

### 1.2 对 deepresearch 的整体启示

- **定位层面**：与其对标单一产品，不如切入“跨场景的深度调研工作流”（学术 + 投研 + 行业情报），在 **多源信息聚合 + 结构化输出 + 报告工作流** 上形成差异。
- **信息源层面**：
  - 竞品多依赖：学术数据库（CrossRef、PubMed、Semantic Scholar 数据、出版社合作）、公开网页、新闻/报告/监管披露、专有商业数据库等。
  - deepresearch 可在：**中文学术与产业报告、本地政策文件、研报、企业公告** 等信息源的系统整合上形成优势。
- **体验层面**：将 **对话式交互（Perplexity）+ 结构化表格视图（Elicit）+ 图谱视图（Connected Papers）+ 证据型引用（Scite/Consensus）** 组合成一个端到端调研工作台。

## 二、学术检索 / 学术问答 / 文献研究

### 2.1 Perplexity Pro（科研与深度检索场景）

1. 市场定位
   - **目标用户群体**：
     - 广义知识工作者：科研人员、数据分析师、投研与咨询顾问、技术/产品/战略角色等。
     - 年龄集中在 20–40 岁，有一定英文阅读能力，乐于尝试新工具。
   - **信息源（核心关注点）**：
     - 公开网页（新闻、博客、技术文档、Wiki 等）。
     - 部分学术文献源（期刊网页、预印本、学术搜索结果等），但并非深度绑定单一学术数据库。
     - 模型自身的通用知识与对网页的抽取能力。
   - **解决的问题与价值主张**：
     - 帮用户节省“自己打开十几个网页比对信息”的时间，通过多源检索 + 总结给出 **可追溯、结构化的回答**。
     - 支持连续追问和深度挖掘，适合在陌生领域快速搭建认知框架。

2. 盈利方式
   - **Pro 订阅制**：按月/年收费，提供更高的查询额度、更强模型、更快响应、文件上传和 Focus/Deep Search 等高级功能。
   - 可能存在团队/企业方案，通过席位或 API 授权收费。

3. 核心功能对比
   - **关键功能清单**：
     - 多源 Web 检索 + AI 聚合回答，附引用链接。
     - Focus / Deep Search：对一个问题进行更深层、多轮的检索与推理。
     - 文件/链接导入后基于内容问答与总结。
   - **产品差异点**：
     - 相比传统搜索：提供直接答案和结构化总结，而不仅是链接列表。
     - 相比纯聊天模型：强化隐式搜索与引用，降低“瞎编”风险。
   - **核心亮点**：高质量答案 + 强可用性的界面 + 快速更新功能。
   - **独特卖点 & 解决的痛点**：
     - 核心卖点：**“AI+搜索的一站式问答”**，用户不必频繁切换网站。
     - 主要解决：信息过载、碎片信息难以整合的问题。
   - **对 deepresearch 的差异机会**：
     - Perplexity 更偏通用搜索与轻量研究，deepresearch 可主打 **深度调研工作流**：项目管理、多轮查阅、证据库沉淀、报告模版等。

4. 市场份额
   - **市场占有率**：在 AI 搜索子赛道处于头部位置，媒体与开发者社区讨论度高。
   - **用户增长趋势**：近两年增长非常快，是“AI 驱动检索崛起速度最快”的代表之一。

5. 服务质量
   - **客服与售后**：主打在线帮助中心、FAQ 和工单/邮件支持，企业方案提供更优支持。
   - **服务覆盖率与附加价值**：功能更新频率高，附带教程、博客和社区讨论。

6. 用户体验
   - **交互设计**：对话框 + 搜索框体验，输入方式自然；结果中突出引用与来源。
   - **易用性**：零门槛，类似“聪明搜索引擎”；自动推荐后续追问降低思考成本。
   - **视觉风格**：简洁工具风，强调可读性和信息密度。
   - **痛点**：
     - 学术数据库深度与覆盖不及专门学术工具；
     - 中文学术与本地行业文档覆盖不足。
   - **爽点**：
     - “一问多源汇总”的体验极佳；
     - 在短时间内搭建基本认知轮廓。

7. 其他信息
   - 对 deepresearch 可作为“通用搜索上游”，deepresearch 可以在其之上定义更长链路、更精细的调研工作流。

### 2.2 Elicit

1. 市场定位
   - **目标用户群体**：系统综述作者、研究生和科研人员，尤其是医学、心理学等需要大量 evidence synthesis 的学科。
   - **信息源（核心关注点）**：
     - 学术文献数据库（如 Semantic Scholar/其他论文索引）中的元数据和部分全文摘要。
     - 侧重英文论文，关注“方法、样本、结论”等核心元素抽取。
   - **解决的问题与价值主张**：
     - 将文献检索 + 关键信息抽取 + 表格整理流程自动化，大幅节省系统综述前期工作时间。

2. 盈利方式
   - 免费基础套餐 + 付费高级功能/额度（订阅或按用量）。
   - 面向机构可能有团队版与教育折扣。

3. 核心功能对比
   - **关键功能清单**：
     - 根据自然语言问题检索相关论文。
     - 自动从论文中抽取结构化字段（干预、样本量、测量指标、结论等）。
     - 支持筛选、排序、导出成 csv/表格，用于系统综述。
   - **产品差异点**：
     - 与通用问答工具相比，强调 **结构化表格视图** 与科研工作流贴合度。
   - **核心亮点**：
     - 对系统综述高度友好，能快速搭建 evidence table。
   - **独特卖点 & 解决的痛点**：
     - 极大缓解了“逐篇读摘要—填 Excel”的枯燥重复劳动。
   - **对 deepresearch 的启示**：
     - 可拓展到“多源报告/研报/政策文件”的结构化抽取和对比表，而不仅是学术论文。

4. 市场份额
   - 在 AI 学术助手细分市场中口碑好，增长较快；
   - 用户规模小于 Google Scholar 等基础设施，但在科研推特/论坛常被提及。

5. 服务质量
   - 有帮助文档、FAQ、简短教程；
   - 注重收集科研群体的反馈，迭代节奏相对稳健。

6. 用户体验
   - **交互设计**：核心界面就是论文表格，默认以“综述工作表”的心智呈现。
   - **易用性**：对科研人员来说自然，对普通用户略有门槛（需理解纳入/排除等概念）。
   - **视觉风格**：偏工具化，信息密度高但清晰。
   - **痛点**：中文与非英文文献支持薄弱，领域覆盖有限。
   - **爽点**：一眼看清一批论文的关键变量与结论。

7. 其他信息
   - 可作为 deepresearch 在“结构化抽取 + 表格视图”方面的 UI/交互参考样本。

### 2.3 Semantic Scholar

1. 市场定位
   - **目标用户群体**：全球科研人员、学生、工程师等需要查找和跟踪学术论文的人。
   - **信息源（核心关注点）**：
     - 来自各大出版社、预印本平台（如 arXiv）、学术机构的论文元数据与部分全文索引。
     - 强调**引文网络、主题标签**等结构化信息。
   - **解决的问题与价值主张**：
     - 在保持免费前提下，提供高质量学术搜索和论文推荐，帮助用户发现相关工作而不仅是关键词匹配。

2. 盈利方式
   - 主要由研究机构/基金支持，产品自身以免费服务学术社区为主。

3. 核心功能对比
   - **关键功能清单**：
     - 按主题、作者、期刊、年份等检索论文。
     - 推荐相关论文，跟踪作者与引用网络。
     - 显示关键概念、影响力指标等。
   - **核心亮点**：
     - 免费 + 覆盖广 + 引文网络能力。
   - **独特卖点 & 解决的痛点**：
     - 减少“只依赖单一搜索词”导致的论文遗漏，通过推荐和网络结构帮助发现相关工作。

4. 市场份额
   - 学术界广泛使用和引用，是 Google Scholar 之外的重要学术搜索入口。

5. 服务质量
   - 提供稳定的 API 与搜索服务，作为学术基础设施存在。

6. 用户体验
   - **交互设计**：传统搜索结果列表 + 论文详情页 + 引用/被引信息。
   - **痛点**：不直接解决“阅读/总结/报告”问题，需要用户自己消化。

7. 其他信息
   - deepresearch 可以在此之上提供阅读理解与工作流功能，而不是与其竞争搜索入口本身。

### 2.4 Consensus

1. 市场定位
   - **目标用户群体**：科研人员、政策制定者、医疗健康从业者、需要“循证决策”的知识工作者。
   - **信息源（核心关注点）**：
     - 学术论文数据库，侧重于从摘要/结论部分中抽取 **“研究结果方向”** 信息。
   - **解决的问题与价值主张**：
     - 将“多篇文献的结论方向”汇总为简明问答，让用户快速知道“证据总体上支持还是反对某观点”。

2. 盈利方式
   - 免费基础版 + 付费高级版（更大额度、更快速度、更丰富功能）。

3. 核心功能对比
   - **关键功能清单**：
     - 用户以自然语言提问，返回答案摘要 + 支持该答案的论文列表。
     - 标记研究结果方向（支持/不支持/混合）。
   - **核心亮点**：
     - 强调“证据方向”而非仅仅给出论文列表。
   - **独特卖点 & 解决的痛点**：
     - 帮助用户快速把握“现有证据的共识是什么”，解决信息积累但结论难以梳理的问题。

4. 市场份额
   - 在 AI 学术问答细分赛道中增长明显，但整体体量仍较小。

5. 服务质量
   - 提供在线帮助与反馈渠道，面向机构用户可提供更个性化服务。

6. 用户体验
   - **交互设计**：搜索框 + 答案卡片 + 文献列表；强调“结论句 + 引用”。
   - **爽点**：一眼看清证据倾向。
   - **痛点**：学科覆盖与更新频率有限，中文领域支持不足。

7. 其他信息
   - deepresearch 若切入政策/医学/公共管理等场景，可借鉴其“结论+证据”呈现方式。

### 2.5 Google Scholar + 学术助手插件

1. 市场定位
   - **目标用户群体**：全球绝大部分科研人员与学生，是事实上的学术检索基础设施。
   - **信息源（核心关注点）**：
     - 各期刊出版社网站、预印本平台、学位论文库等；以大规模网页索引为基础。
   - **解决的问题与价值主张**：
     - 提供免费、广覆盖的论文检索与引用信息查询。

2. 盈利方式
   - 核心搜索免费，间接为 Google 生态导流，不以该产品本身营收为主。

3. 核心功能对比
   - **关键功能清单**：
     - 论文检索、引用次数查询、相关论文推荐。
     - 与学术助手插件配合可实现摘要生成、引用导出、PDF 一键打开等能力。
   - **产品差异点**：
     - 其本身更像“入口”，AI 辅助功能依赖外部插件组合。

4. 市场份额
   - 在学术搜索领域占据绝对主导，几乎所有科研用户都会使用。

5. 服务质量
   - Google 级别的基础设施，稳定性好；定制支持有限。

6. 用户体验
   - **交互设计**：极简搜索框 + 列表结果；
   - **痛点**：不处理阅读、总结、对比；用户需要自己做二次整理。

7. 其他信息
   - deepresearch 可与之形成“入口 + 深度分析层”的关系，兼容其链接生态。

### 2.6 Connected Papers / Research Rabbit

1. 市场定位
   - **目标用户群体**：需要“快速认识一个领域”的科研人员、博士生、技术从业者。
   - **信息源（核心关注点）**：
     - 学术论文数据库中的引文与共同引用关系；
     - 关注“论文之间的网络结构”而非单篇内容。
   - **解决的问题与价值主张**：
     - 帮用户通过图谱快速理解某领域的核心论文、分支与演化路径。

2. 盈利方式
   - 免费基础版 + 订阅制高级功能（更多图谱项目、更大存储、团队协作）。

3. 核心功能对比
   - **关键功能清单**：
     - 输入一篇或若干篇论文，生成文献关系图谱。
     - 支持图谱浏览、节点细节查看、收藏与分享。
   - **核心亮点**：
     - 可视化的“文献地图”，适合探索式学习和早期扫盲。
   - **独特卖点 & 解决的痛点**：
     - 解决“只看搜索结果列表无法把握整体结构”的痛点。

4. 市场份额
   - 在文献可视化细分领域知名度高，但整体市场较小。

5. 服务质量
   - 小团队产品，以在线帮助与社区反馈为主。

6. 用户体验
   - **交互设计**：图谱拖拽、缩放、节点点击查看详情。
   - **痛点**：
     - 新手需要时间理解图谱含义；
     - 中文文献支持有限。
   - **爽点**：看到“整个领域的地图”。

7. 其他信息
   - deepresearch 可考虑将“图谱视图”与文本/表格视图结合，而不是只做可视化。

---

### 2.7 Scite

1. 市场定位
   - **目标用户群体**：科研人员、期刊编辑、基金会和评估机构。
   - **信息源（核心关注点）**：
     - 来自期刊和数据库的引文信息与引用上下文。
     - 重点在“引用句子和语义标签（支持/反驳/中立）”。
   - **解决的问题与价值主张**：
     - 提供“证据型引用”视角，避免只看引用次数却不了解引用性质。

2. 盈利方式
   - 个人与机构订阅制 + API 授权。

3. 核心功能对比
   - **关键功能清单**：
     - 标注引用类型（支持/反驳/中立）。
     - 展示引用片段的上下文。
   - **核心亮点**：
     - 在引用语义上的深度加工，强调证据质量而非数量。
   - **独特卖点 & 解决的痛点**：
     - 帮研究者迅速找到“支持或反对某结论”的关键文献。

4. 市场份额
   - 在生命科学和医学等领域有一定影响力，常被提及。

5. 服务质量
   - 提供 API、机构合作与技术支持。

6. 用户体验
   - **交互设计**：在论文详情中突出证据型引用模块，支持筛选。

7. 其他信息
   - deepresearch 可以借鉴其“观点—证据—反证”的结构化框架，扩展到非学术文档。

## 三、通用 AI 搜索与总结

### 3.1 Perplexity Pro（通用场景补充）

- 已在 2.1 展开，此处补充：
  - 在新闻、博客、技术文档等通用内容上表现优异，可视为“通用信息层”。
  - deepresearch 可定位在“更深的研究层”，专注多文档对比、结构化洞察与报告产出。

### 3.2 Kagi Universal Summarizer

1. 市场定位
   - **目标用户群体**：偏技术和信息重度用户，追求高质量、无广告体验的用户。
   - **信息源（核心关注点）**：
     - 用户提供的网页、PDF、文章等 URL 内容。
     - 关注“从单个页面快速抽取关键信息”。
   - **解决的问题与价值主张**：
     - 帮用户减少对长网页逐字阅读的时间，通过一键摘要快速预览是否值得深读。

2. 盈利方式
   - 作为 Kagi 搜索生态的一部分，以订阅制为主（整体搜索订阅中包含总结功能）。

3. 核心功能对比
   - **关键功能清单**：
     - 输入链接，一键生成摘要和要点列表。
     - 提供多种摘要风格（简短 TL;DR、详细概要等）。
   - **产品差异点**：
     - 强调轻量、高质量的单页总结，而不是全流程研究。
   - **核心亮点**：
     - 极简操作 + 高质量摘要。
   - **独特卖点 & 解决的痛点**：
     - 解决“打开文章但没时间读”的痛点，作为信息过滤层。

4. 市场份额
   - 主打小众高端市场，用户数量不大但粘性强。

5. 服务质量
   - 小团队 + 社区驱动，响应较快。

6. 用户体验
   - **交互设计**：极简界面，只需输入 URL 即可；
   - **爽点**：几秒内获得可读摘要。

7. 其他信息
   - 对 deepresearch 的启示：
     - 单页摘要可以作为研究起点，但核心价值应在“多文档综合 + 结构化洞见”。

## 四、企业情报 / 投研研究

### 4.1 CB Insights

1. 市场定位
   - **目标用户群体**：投资机构、咨询公司、企业战略/创新部门。
   - **信息源（核心关注点）**：
     - 公司注册信息、融资事件、新闻报道、专利数据等多源商业数据。
     - 强调“公司与行业事件”的结构化数据库。
   - **解决的问题与价值主张**：
     - 帮助用户做行业扫描、技术趋势分析、并购/投资机会识别。

2. 盈利方式
   - 高价企业订阅制，按席位或 license 收费。

3. 核心功能对比
   - **关键功能清单**：
     - 公司与融资数据库、市场地图、行业报告和可视化图表。
   - **产品差异点**：
     - 数据深度与行业覆盖广，尤其在初创/创新方向。
   - **核心亮点**：
     - 可视化的市场地图与专题报告。
   - **独特卖点 & 解决的痛点**：
     - 大幅缩短企业情报收集时间，是高层决策常用工具。

4. 市场份额
   - 在企业情报与创新研究领域属于主流方案之一，与 PitchBook 等相互补充。

5. 服务质量
   - 提供客户成功团队、培训与定制服务。

6. 用户体验
   - 仪表盘式界面，多图表与筛选器；
   - 对专业用户友好，但学习成本不低。

7. 其他信息
   - deepresearch 可定位为“轻量/开放数据版情报研究工具”，而非重资产数据提供商。

### 4.2 AlphaSense

1. 市场定位
   - **目标用户群体**：投研机构、企业战略部门、IR、法律与合规团队。
   - **信息源（核心关注点）**：
     - 研报、财报、电话会议纪要、新闻、法规文档等。
     - 重点在“文档内语义搜索与高亮”。
   - **解决的问题与价值主张**：
     - 在海量专业文档中快速找到关键信息，支持投研与决策。

2. 盈利方式
   - 企业订阅制，根据用户数量和文档库范围定价。

3. 核心功能对比
   - **关键功能清单**：
     - 语义搜索跨多类文档；
     - 命中高亮、保存搜索、告警通知、研究笔记。
   - **核心亮点**：
     - 与投研工作流程深度融合。
   - **独特卖点 & 解决的痛点**：
     - 解决“文档太多、找不到关键句”的痛点。

4. 市场份额
   - 在欧美投研圈采用度高，是主流情报解决方案之一。

5. 服务质量
   - 企业级部署与支持，客户成功与培训服务完善。

6. 用户体验
   - 以文档阅读视图为核心，强调上下文和高亮；
   - 对非专业用户略复杂。

7. 其他信息
   - deepresearch 可在此基础上延伸“多文档综合分析与自动化报告”能力。

## 七、对 deepresearch 的综合策略启示

1. **市场定位与目标用户**
   - 目标人群：重度知识工作者（研究生/研究员/分析师/咨询顾问/产品与战略人员等），其共性是：需要频繁写“结构化研究输出”（报告、备忘录、分析简报）。
   - 定位建议：做“跨学术 + 投研 + 行业情报”的 **深度调研工作台**，而不是单一搜索或单一数据终端。

2. **信息源策略**
   - 聚焦：多源公开数据（学术、新闻、政策文件、企业公告、研报摘要、行业白皮书等）。
   - 差异化：在 **中文与本地化文档** 支持上形成优势，构建“中英文混合调研能力”。

3. **核心功能与差异点**
   - 从“问答”向“工作流”升级：
     - 选题/问题定义 → 多源检索 → 多文档阅读与标注 → 结构化提炼 → 报告生成 → 引用与版本管理。
   - 视图层：
     - 对话视图（像 Perplexity），
     - 表格视图（像 Elicit），
     - 图谱视图（像 Connected Papers），
     - 证据对立视图（借鉴 Scite/Consensus）。

4. **盈利方式**
   - 个人：基础免费 + Pro 订阅（更大额度、更长上下文、更多项目空间、多视图功能）。
   - 团队/机构：席位制 + 协作空间 + 权限与合规；可逐步增加企业功能。

5. **服务质量与用户成功**
   - 提供针对具体场景的教程（如何用 deepresearch 完成一篇综述/一篇行业报告/一份投资备忘录）。
   - 通过社区与案例沉淀 best practice，帮助用户从“工具使用”迁移到“方法论升级”。
